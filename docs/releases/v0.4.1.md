# ocbrain v0.4.1

This patch makes the local core's operating model fail-closed and explicit.

Hosted judge and embedding calls now default disabled in source configuration;
credentials alone cannot activate them. Hosted-teacher request packaging has a
separate default-disabled authority flag and still never dispatches a call.

Dataset mining, classification, local grading, selection, export, and the human
audit path remain available. Pilot preparation and training-result recording
return `blocked` by default. They require both the completed named-human audit
and a separate local `dataset.training_enabled=true` opt-in.

The former light autopilot, heavy autopilot, and stallcheck launchd jobs are
retired. Their plist labels remain only to identify legacy installations; each
artifact is disabled, has no timer, and executes `/usr/bin/false`. Maintenance
profiles and stall diagnostics are manual compatibility tools, not background
services.

MCP remains a local, on-demand process. Each runtime must be registered
explicitly, and clients must start a fresh process after an upgrade; an existing
task can retain the older process it already started.
