# OCBrain v1.0.0 — One event-authoritative core

v1 removes the old two-plane database from the default product. The append-only
event chain is the single semantic authority; evidence, beliefs, provenance
links, aliases, and FTS are deterministic projections. Retrieval, source-handle,
egress, and closeout tables remain append-only operational receipts.

## Strict core

- exact allowlisted schema with no relational legacy, dataset, trainer, judge,
  embedding, autopilot, scheduler, watchdog, deadman, or paging tables;
- preserved legacy event sequence and hashes, including non-contiguous rowids;
- corruption refusal rather than silent fallback;
- deterministic import events for relational evidence, knowledge, links,
  signals, lifecycle constraints, and retrieval snapshots;
- conservative legacy scope and stable aliases for old IDs;
- body hashes kept separate from historical source-file hashes;
- bulk deterministic FTS rebuild with integrity checking;
- full semantic projection rebuild without deleting live retrieval or closeout
  receipts.

## Shared intelligence receipts

`ocbrain.closeout.v1` now supports optional `ocbrain.action.v1` and
`ocbrain.outcome.v1` envelopes. Actions preserve mechanism plus local semantic
role, target, context, policy/model, cost, provenance, and versioned features.
Outcomes preserve metric vectors, baselines, counterfactuals, attribution,
uncertainty, verifier evidence, and local interpretation rather than collapsing
unlike environments into one scalar reward.

## Optional packages

- `ocbrain` — strict core and MCP;
- `ocbrain-training` — isolated local curation/audit/training preparation;
- `ocbrain-ops` — isolated manual diagnostics and legacy operations.

Companions use their own SQLite databases, are absent from the core wheel and
MCP import graph, and install no recurring job. Legacy companion mutation
requires an explicit legacy database.

## Migration

`core-migrate-v1` is archive-first and fresh-path-only. It produces a verified
archive, strict core, training extract, ops extract, and manifest. It never
modifies or repoints the live source. Activation uses a separate ignored local
pointer after verification.

## Acceptance gate

Release acceptance requires the full source suite, Ruff and package inventory,
clean-environment core/companion installs, live migration artifact verification,
projection rebuild equivalence, and real fresh Codex, Claude Code, and OpenClaw
`context → source → feedback → closeout` turns against the same core. No hosted
call, timer, training run, watchdog, or automatic activation is allowed during
those checks.

The gate passed on 2026-07-13: 632 tests, Ruff, package isolation, full
migration/refold verification, and fresh Codex, Claude Code, and OpenClaw
round trips all passed. Post-acceptance SQLite integrity and foreign-key checks
also passed, so the provisional activation pointer was retained.
