# OCBrain v0.5.0 — Shared Context

v0.5 introduces the stable cross-client contract that sits between an
unbounded local evidence lake and a model's bounded attention.

## Runtime contract

- `brain.context` returns `ocbrain.context.v1` with resolved scope, ranked
  serving items, contradictions, exclusions, coverage, and source handles.
- `brain.source` expands only an issued handle in its original scope, within a
  character bound, after content-hash verification.
- Source issuance history is append-only and records every retrieval that
  received the handle.
- `brain.feedback` records usefulness on an issued retrieval ID.
- `brain.closeout` appends `ocbrain.closeout.v1` with task status, decision
  impact, artifacts, verifiers, provenance, and retrieval links.
- Closeouts update the linked retrieval's decision-impact field without
  promoting a belief.

## Authority

The default runtime profile exposes eight tools: context, source, search,
digest, get, feedback, ingest, and closeout. Protected correction, proposal,
preview, and tombstone operations require the admin profile.

`--allow-writes` is corrected from a historical no-op into a deprecated alias
for `--profile admin`. Ordinary client registrations remain runtime-only.

No hosted call, training run, scheduler, pager, or automatic database activation
is part of Shared Context.

## Verification

The combined legacy/v1 Shared Context and MCP contract passes its focused suite,
including scope/hash refusal, handle reissuance, feedback linkage, append-only
closeouts, runtime/admin separation, and strict v1 source round trips.
